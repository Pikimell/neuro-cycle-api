# Додаткові ендпоінти для покриття вимог README (MVP)

Нижче — перелік REST-ендпоінтів, яких поки немає або вони не описані як готові, але потрібні для покриття описаних сценаріїв у README.

- **Auth / Access Control**
  - `POST /auth/me` — повертає профіль поточного користувача за access token (використовується для автологіну після відкриття додатку).

- **Mood / Daily Survey**
  - `POST /mood-entries/daily` — зберегти сьогоднішній настрій (оцінка + коментар), з перевіркою що за день лише одна основна точка або з політикою перезапису.
  - `GET /mood-entries/current` — отримати останній зафіксований стан для головної картки.
  - `GET /mood-entries/history?from&to` — добова історія для статистики/календаря.

- **Recommendations / Tasks**
  - `GET /recommendations` — список рекомендацій для користувача без діагнозу (можливо з урахуванням останнього настрою).
  - `GET /questionnaire-templates/assigned` — опитування, які треба пройти користувачу (визначення за діагнозом/правилами).

- **Migraine Attacks**
  - `POST /migraine-attacks/:id/complete` — оновити тривалість/поля після завершення нападу (коли створено під час нападу).
  - `GET /migraine-attacks/calendar?month&year` — список дат/підсумків для календаря.

- **Diary (Day View)**
  - `GET /diary/day?date=YYYY-MM-DD` — агреговані дані за день (настрій, напади, нотатки, нагадування, відповіді на опитування).

- **Medication Reminders**
  - `POST /medication-reminders/:id/mark-taken` — позначити дозу як прийняту (оновити `lastTakenAt`, опційно час/дата).
  - `POST /medication-reminders/:id/mark-missed` — позначити як пропущену (для статистики/нагадувань).

- **Appointments**
  - `POST /appointments/:id/respond` — лікар підтверджує/відхиляє/переносить (body: status, scheduledStart/End, comment).
  - `GET /appointments/my` — отримати записи, релевантні авторизованому користувачу (пацієнту або лікарю).

- **Doctor Info**
  - `GET /doctors/me` — дані лікаря, якщо користувач — DOCTOR.
  - `GET /doctors/:id/availability` — робочі години/слоти для запису (MVP може віддавати текстовий розклад).

- **Questionnaires**
  - `POST /questionnaire-responses` — збереження відповіді пацієнта (шаблон + відповіді).
  - `GET /questionnaire-responses/my?from&to` — історія пройдених опитувань користувача.

- **Visibility / Access Rules**
  - `GET /patients/me` — швидкий доступ до профілю пацієнта для поточного користувача.
  - `GET /patients/:id/summary` — для лікаря: базовий профіль + клінічні блоки (алергії, застереження, хронічні стани) з урахуванням прапорів видимості.

За потреби частина цих ендпоінтів може бути об’єднана або реалізована через параметри вже існуючих CRUD, але вони виділені окремо для підтримки сценаріїв, описаних у README.*** End Patch
